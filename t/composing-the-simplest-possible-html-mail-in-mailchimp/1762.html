After reading your guide I did some testing to see what HTML Gmail generates when using their web cilent formatting tools. I used the ‚ÄúShow original‚Äù option to view the raw email. I‚Äôm sharing the findings here in case its of interest:

  • The Gmail web client didn‚Äôt generate p tags. Instead of p tags, it used divs. Divs that look like paragraphs are separated by another div that contains only a br tag.
  • There are four sizes of text available in the formatting tools: ‚ÄúHuge‚Äù, ‚ÄúLarge‚Äù, ‚ÄúNormal‚Äù, and ‚ÄúSmall‚Äù. Text is formatted with one of:

  • Text color is set by wrapping with font tag and using the color="‚Ä¶ (e.g. #ff0000)" attribute.

At the same time, I sent this email from the Gmail web cilent to Mailchimp to see if they would alter the HTML at all when importing it into a campaign. The changes I noticed:

  • The fake-paragraph-separators changed from a div-wrapped br tag to div-wrapped nbsp character entity.

  • Mailchimp added the mandatory footer with unsubscribe links etc.
