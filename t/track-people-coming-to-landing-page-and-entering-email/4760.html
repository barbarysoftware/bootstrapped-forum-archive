unboot February 5, 2017, 3:00pm #2

Avoiding js is easy, if you can do a little work “by hand”.
Install https://wordpress.org/plugins/redirection/ , then make a new redirection target for each source you have, e.g yoursite.com/from-adwords/ and redirect this through the plugin to your landing page. The plugin keeps the statistics. You then have to put in adwords as a target url your /from-adwords path.
Another similar idea is to go through an url shorter service, but I think it’s better to keep this “in site”.

Clay_Nichols February 5, 2017, 11:57pm #4

I THINK Mixpanel does not require you to write .js (or if you do, it’s a very simple snippet). I did it and I didn’t need any .js knowledge.

Here’s how to do it in WP
(I’ve not tested the code below, don’t have access to my site right now, but this looks like what I used)
mixpanel.track(‘Page View’, {
‘Page Title’ : document.title,‘URL’ : window.location.pathname,‘Page Type’:‘Page’
});

You can just include their .js snippet, much like Adwords.

They also have something new: Autotrack
Here’s how to autotrack all pageviews (I’ve not needed it, so not tried it yet)
https://mixpanel.com/help/questions/articles/how-do-i-track-page-views-with-autotrack
