After interviewing candidates for various roles in the past, I found that just asking questions about skills, discussing candidateâ€™s experience etc. is usually not that useful, as candidates often overstate their abilities. In the end, working on a few typical tasks of the role provides a much better and faster way to see if a candidate is a good fit.

E.g. with a support position you might want to answer some typical customer requests together, troubleshoot a common a problem, maybe do a customer call etc. For a developer position you might want to work on a small project as part of the interview process, do some architecture design, pair programming and so on.
